# Make Repository Public

Use these steps only for the clean snapshot repository. Do not change the private development repository visibility.

1. Open GitHub repository settings.
2. Go to General.
3. Scroll to Danger Zone.
4. Select Change repository visibility.
5. Choose Make public.
6. Confirm the repository has no current tracked secrets.
7. Confirm Actions logs are acceptable for public visibility.
8. Confirm Issues and Pull Request templates are present.
9. Confirm Release notes have no private network data, local personal paths, Apple credentials, or app-specific passwords.
10. Add repository topics from [github-topics.md](github-topics.md).

After a private repository becomes public, source code, Actions logs, activity, and fork capability become public.
