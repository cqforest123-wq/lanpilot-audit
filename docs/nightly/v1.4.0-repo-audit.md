# LANPilot Audit v1.4.0 Repository Audit

## Current Functions

- macOS Tauri application with explicit authorization, engine setup, interface confirmation, fixed-step audit execution, localized reports, governance views, remediation metadata, and ZIP export.
- Bundled CLI engine performs low-intensity, point-in-time governance observations and structured report generation.

## Scripts

- Quality: `check`, `i18n:visible-check`, `i18n:strict-check`, `security:check`, `security:deep-check`, `qa:governance-fixtures`, `qa:installed-app`.
- Release: `release:local`, `release:website`, `release:verify`, `release:remote-check`, `gatekeeper:check`, `release:artifacts:verify`, `release:smoke`, `release:prepare`.
- Integrity: `version:check`, `engine:sync:check`, `workflow:check`, `checksums`.

## Allowlisted Step IDs

`init_lab`, `baseline`, `passive_assets`, `client_isolation`, `common_services`, `smb_posture`, `gateway_posture`, `build_report`, `local_network_config`, `mdns_observation`, `web_tls_baseline`, `build_enhanced_governance_report`, `build_formats`.

## Report Outputs

- Executive summary, technical report, evidence index, risk register, remediation roadmap.
- Asset inventory, service exposure matrix, local network configuration, mDNS observations, Web baseline, TLS certificates.
- Snapshot difference, remediation tracking, governance summary, Markdown/HTML/CSV/JSON/XLSX artifacts and ZIP export.

## GitHub Workflows

- `ci.yml`: main/tag/pull-request quality and security gates.
- `release.yml`: manually triggered local release smoke workflow; it does not publish GitHub releases.
- `public-release.yml`: manually triggered Developer ID signing and notarization path when credentials are available.

## Security Boundary

- Rust accepts enum-based fixed steps only; authorization is consumed before real checks.
- Engine path, scripts, output locations, and command arguments are fixed and integrity checked.
- Read-only observation, report reading, local remediation metadata, and local ZIP export only.
- No arbitrary command input, credential activity, unauthorized login, configuration mutation, lateral movement, or cloud upload.

## i18n

English, Simplified Chinese, Traditional Chinese, Japanese, Korean, German, French, Spanish, Brazilian Portuguese, Italian, and Dutch.

## Release Artifacts

macOS `.app`, Apple-silicon DMG, SHA256SUMS, local release notes, release website, GitHub pre-release assets, Gatekeeper report, and QA reports.

## Known Issues

- Developer ID Application certificate and notarization credentials are unavailable; internal DMG remains ad-hoc signed and not notarized.
- Raw evidence remains in its original language by design.
- Findings are point-in-time observations and depend on the current local network view.

## High-Risk Features Not Recommended

Do not add arbitrary shell or custom commands, credential testing, vulnerability exploitation, directory fuzzing, unauthorized login, network configuration changes, automatic remediation, lateral movement, or cloud evidence upload.
