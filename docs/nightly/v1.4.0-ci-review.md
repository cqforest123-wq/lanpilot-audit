# v1.4.0 CI and Release Review

- Main and tag CI run the full quality gate, visible localization check, and deep security boundary check.
- Tag workflow builds and verifies the DMG, generates checksums, performs launch smoke testing, publishes a pre-release, verifies the published release, and uploads CI artifacts with pinned `actions/upload-artifact` v4.6.2.
- Apple credentials are not stored in the repository.
- Internal releases explicitly state ad-hoc signing, no notarization, internal-testing status, and pending Developer ID.
- Public signing and notarization remain a separate manual workflow requiring repository secrets.
